# Sqllogictest smoke manifest — the curated list of `.slt` files that the
# `slt_smoke` test runs by default. Paths are relative to the corpus root
# (the `target/slt/` directory populated by `xtask/fetch-slt.sh`).
#
# Set `RUSTQLITE_FULL_SLT=1` to also exercise every file in this manifest
# (today: all of them; the env var is wired for a future sub-corpus gate).
#
# The M5.1 milestone introduces a write-path + index: the most useful smoke runs are the
# `evidence/` files (small, fast, exercise DML/DDL with the engine we just wrote) plus
# the in-tree `our/*.slt` files. Files that depend on un-shipped features
# (CREATE INDEX over partial-index WHERE, REPLACE conflict handler, REINDEX, triggers,
# views, CASE/WHEN expressions, subqueries, UPSERT, multi-column indexes, expression
# indexes) are excluded; they fail today and are tracked in the M5+ roadmap.
#
# As each M5+ feature lands, the matching evidence/ test(s) is added here. The full
# upstream corpus (~5,000 .slt files in `random/` and `select{2,3,4,5}.test`) is
# wired through `xtask/fetch-slt.sh` and the test will exercise it when M5+ lands.
#
# In-tree files: a path beginning with `our/` is read from the repo's own
# `crates/rustqlite/tests/slt/` directory. Use this for hand-rolled smoke
# files that exercise what the engine ships without depending on the
# upstream `xtask/fetch-slt.sh` corpus.

# M5.2.10 — expression indexes (CREATE INDEX on arbitrary expressions, maintenance,
# UNIQUE constraint semantics).
our/expression-index.slt

# M5.0 — `UPDATE`. The upstream `evidence/slt_lang_update.test` mixes
# `CREATE INDEX` (still on the M5+ roadmap) and `skipif`/`onlyif`
# directives the harness's parser does not yet understand, so the hand-
# rolled `our/update.slt` covers the same UPDATE surface against our own
# engine. The full file lands in the manifest when both gaps close.
our/update.slt

# M5.1 — single-column indexes + DROP INDEX + indexed-equality SELECT.
our/index.slt

# M5.2 — multi-column indexes + prefix equality SELECT + composite-key maintenance.
our/multi-column-index.slt

# M5.1 — `DROP INDEX` from the upstream evidence corpus. Small (~75 lines),
# exercises CREATE/DROP INDEX without the partial-index / multi-column /
# skipif-onlyif dependencies that the rest of the slt_lang_*.test family
# rely on.
evidence/slt_lang_dropindex.test
